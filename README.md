# 方案
微调还是RAG方案
基于此问题，RAG方案有两个问题：
1.题目数量多，每道题是一个单独的节点，效率低；
2.题库的问题比较专业，base模型不能正确的理解问题需求
故此方案选择微调

模型训练状态选择拟合还是过拟合
因为题库不会频繁更新，对精度要求高，此项目偏向于过拟合

训练集测试集如何划分
因为训练偏向于过拟合，所以训练选择全部的训练集，测试从训练集中随机筛选一部分


# 模型
硬件条件，现有硬件条件支持1.5B-7B的大模型微调
选择基准：模型越大，直观的说越聪明，微调的难度越高；针对当前项目的数据量，选择一个较小的模型和一个较大的LORA模型，更容易达到效果

# 训练与评测
基础模型 deepseek-ai/DeepSeek-R1-Distill-Qwen-1.5B
lora参数：秩32，缩放系数64，截断长度660 批大小8
